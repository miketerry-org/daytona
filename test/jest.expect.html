Matcher Description / usage
toBe(value) Exact identity / primitive comparison using Object.is
toEqual(value) Deep equality (objects, arrays) comparison
toStrictEqual(value) Strict deep equality (including types, no extra properties)
toBeNull() Matches only null
toBeUndefined() Matches only undefined
toBeDefined() Opposite of toBeUndefined
toBeTruthy() Matches any “truthy” value
toBeFalsy() Matches any “falsy” value
toBeGreaterThan(number) Numeric comparison: >
toBeGreaterThanOrEqual(number) Numeric comparison: ≥
toBeLessThan(number) Numeric comparison: <
toBeLessThanOrEqual(number) Numeric comparison: ≤
toBeCloseTo(number, numDigits?) For floating-point approximate equality
toBeInstanceOf(Class) Checks `instanceof` against a class / constructor
toMatch(regexp | string) Matches string or regex against a string value
toContain(item) Checks whether an item is in an array or substring in string
toContainEqual(item) Deep equality version of toContain
toHaveLength(number) Asserts `.length === number`
toHaveProperty(path, value?) Asserts object has property (optionally with specific value)
toMatchObject(object) Asserts that received object “matches” subset of expected properties
toMatchSnapshot() Snapshot assertion (serializes and compares)
toBeCalled() / toHaveBeenCalled() Used with mocks / spies: assert function was called
toHaveBeenCalledWith(...args) Asserts mock/spy was called with specific arguments
toHaveBeenLastCalledWith(...args) Asserts mock/spy’s most recent call had these args
toHaveBeenNthCalledWith(n, ...args) Asserts the n‑th call had given args
toHaveReturned() Asserts the mock/spy returned at least once
toHaveReturnedWith(value) Asserts the mock/spy returned a specific value at least once
toHaveLastReturnedWith(value) Asserts the last call return value matches
toHaveNthReturnedWith(n, value) Asserts the n‑th call’s return value matches
toThrow(error?) Asserts function throws (optionally matching message, class, regex)
toThrowError(error?) Alias for toThrow
resolves Used for promise assertions: wraps `expect(promise).resolves…`
rejects Used for promise rejections: `expect(promise).rejects…`
not Negation modifier (e.g. expect(x).not.toBe(y))
